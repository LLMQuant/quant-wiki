# How to Contribute

The Quant-Wiki project team warmly welcomes your contributions. It is thanks to hundreds of contributors like you that Quant-Wiki has grown to what it is today!

This article outlines the process for contributing to Quant-Wiki. Please read the following guidelines carefully before writing or editing wiki pages to help ensure high-quality content.

---

## Contribution Guidelines

Before editing, please review the [Quant-Wiki Contribution Guide](https://github.com/LLMQuant/quant-wiki/blob/master/.github/CONTRIBUTING.md) and project policies to facilitate effective collaboration with other contributors.

---

## Getting Involved

???+ warning
    Before writing new content, please check the [Issues](https://github.com/LLMQuant/Quant-Wiki/issues) to ensure no one else is working on the same topic. If not, open a new Issue to document what you plan to write.

???+ tip
    The Issues page also contains many bugs and enhancements to address, especially in our Iteration Plans. Picking up tasks from there is an excellent way to start contributing!

To ensure the accuracy and professionalism of all entries, we recommend:

1. **Write about what you know**: Prioritize topics aligned with your professional expertise or academic background.
2. **Be cautious with unfamiliar domains**: If you are still learning about a subject, deepen your understanding before contributing.
3. **Consult authoritative references**: When adding or revising content, refer to reputable sources to ensure accuracy.

---

## Editing on GitHub

Contributing requires a GitHub account ([register here](https://github.com/join)), but does not require advanced GitHub skills.

???+ tip
    Your changes will not appear on the main site until merged into the main repository, so there is no risk of breaking existing content. See the [GitHub contributing guide](https://docs.github.com/en/get-started/quickstart/contributing-to-projects).

### Editing a Single Page

1. Navigate to the relevant page in the Quant-Wiki project;
2. Click the **"Edit this page"** button to be redirected to GitHub;
3. Write or modify content in the editor (disable automatic translation tools);
4. Fill in the commit message and click **Propose changes**;
5. Click **Create pull request** and follow the prompts;
6. Wait for a maintainer to review and merge your changes.
